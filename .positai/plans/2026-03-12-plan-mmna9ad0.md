# Plan: KendallColijn Distance Optimization

## Problem

`KCVector()` (the per-tree vector computation for the KC metric) is the bottleneck.
It uses `apply(combn(tips, 2), 2, function(i) max(intersect(ancestors[[i[1]]], ancestors[[i[2]]])))` —
O(n²·d) R-level set intersections. For 200-tip trees this is ~60× slower than CID.

## Approach: C++ `kc_vector()` function

The existing `path_vector()` in `src/path_vector.cpp` already builds an ancestry matrix
and finds the common-ancestor divergence point for all tip pairs in C++. `KCVector` needs
almost the same thing: instead of `n_ancs[i] + n_ancs[j] - 2*common` (path length), it
needs `common` (MRCA depth from root = number of common ancestors).

### Step 1: Add `cpp_kc_vector()` to `src/path_vector.cpp`

A new Rcpp-exported function modelled on `path_vector()`:

```cpp
// [[Rcpp::export]]
IntegerVector cpp_kc_vector(IntegerMatrix edge, IntegerVector tip_order) {
  // Same ancestry-matrix construction as path_vector()
  // For each pair (tip_order[i], tip_order[j]) with i < j:
  //   find number of common ancestors → that's the MRCA depth
  // Return as dist-order vector
}
```

Key differences from `path_vector()`:
- Accepts `tip_order` (sorted by tip label) so pairs match the R `combn(order(tip.label), 2)` order
- Returns `common` (MRCA depth) instead of path length
- The iteration order must match R's `combn()` column order for the result to be compatible

### Step 2: Update `KCVector()` in R

Replace the R implementation with a call to `cpp_kc_vector()`:

```r
KCVector <- function(tree) {
  tree <- Preorder(tree)
  edge <- tree[["edge"]]
  tipOrder <- order(tree[["tip.label"]])
  nTip <- edge[1, 1] - 1L
  structure(cpp_kc_vector(edge, tipOrder), Size = nTip, class = "dist")
}
```

### Step 3: Optimize `KendallColijn()` all-pairs Euclidean distance

The all-pairs case (line 106–116) currently does:
```r
treeVec <- vapply(tree1, Vector, ...)  # n_pairs × n_trees matrix
apply(combn(N, 2), 2, function(i) .EuclideanDistance(treeVec[,i[1]] - treeVec[,i[2]]))
```

This can use the existing `pair_diff_euclidean()` C++ function instead.
Similarly, the cross-pairs case (lines 119–123) can use `vec_diff_euclidean()`.

- Verify `KendallColijn()` all-pairs produces identical distances

- Edge cases: star tree (1 split), pectinate, balanced, polytomies

## Expected impact

The bottleneck shifts from O(n²·d) R-level `intersect()` calls to O(n²) C++ array
comparisons (same as `path_vector()`). For 200-tip trees, this should give roughly
a 30–60× speedup on the per-tree vector computation, making KC comparable in speed
to PathDist.
